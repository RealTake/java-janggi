
# 기능 구현 목표
- 각 기물들의 움직임을 궁 또는 궁성의 영역 포함하여 구현 한다.
- 장기의 점수를 계산하여 승패와 관계 없이 모두 노출한다.
- DB 적용
  - 애플리케이션을 재시작하더라도 이전에 하던 장기 게임을 다시 시작할 수 있어야 한다.
  - DB를 적용할 때 객체의 변경을 최소화해야한다.

# 의문점
- 코드를 작성함에 있어 간접적으로 알아낼 수 있는 것과 직접적으로 알아낼 수 있는 것. 어떤 것을 선택해야할까?
  - EX) 현재 턴에서 게임이 종료되었다면, 현재 턴을 가지고 있는 팀이 승리자가 될 것이다! 
        그 이유는 자기 자신의 킹은 죽일 수 없기 때문이다
        여기서, 그럼 승리자를 구하는 로직에서 현재 턴인 팀을 가져오면 될까? 아니면 킹이 살아있지 확인 후 해당 팀을 가져와야할까?

## DB 고민점
- Transanction 관리하는 클래스와 도메인 객체가 결합되어도 괜찮을까? 
- DAO에 create, update문 날릴 때 Entity로 소통하고있다.. 고민할 시간이 부족하였다

# 기능 구현 목록
- 각 기물들의 움직임을 궁 또는 궁성의 영역 포함하여 구현 한다.
  - [x] 차, 포는 궁성안의 5가지 좌표에서 대각선으로 이동가능하다.
  - [x] 왕과 사는 1칸씩만 이동 가능하다
  - [x] 왕과 사는 궁성안에서만 움직일 수 있다

- 장기의 점수를 계산하여 승패와 관계 없이 모두 노출한다.
  - [x] 각 기물에 점수를 부여할 수 있다. 부여받지 않는다면 기본값을 가진다
  - [x] 자신의 팀 기물의 총합을 구할 수 있다
