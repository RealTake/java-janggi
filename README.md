# java-janggi

콘솔 입출력으로 진행 가능한 장기 게임을 자바로 구현했습니다.<br>
7가지 종류의 장기 말들을 추상화하기 위해 `Piece` 인터페이스와 이를 구현한 `AbstractPiece` 추상 클래스를 이용했습니다.<br>
각 말을 대표하는 클래스들을 `Piece` 타입으로 추상화해 컴파일 시점의 의존성을 관리하고 런타임에는 `AbstractPiece`를 상속받아 구현한 구체 클래스들에 의존하도록 설계했습니다.<br>
각 말의 움직임과 관련된 로직은 해당 클래스에 구현해둠으로써 말이 자신의 움직임에 대한 규칙을 관리하도록 했습니다.<br>
장기판의 좌표를 나타내는 `Point` 클래스를 구현하고 `Point`와 `Piece`를 참조하는 `Position` 클래스로 말과 좌표상의 위치를 함께 관리하도록 설계했습니다.<br>
이렇게 설계한 이유는 말은 이동하는 책임만 맡을 뿐 위치에 대한 정보를 알 필요는 없으며 장기판의 고정된 위치가 말을 관리하는 것이 맞다고 생각했기 때문입니다.<br>

## 요구사항

- 게임은 초나라(초록색)와 한나라(빨간색)의 대결로 진행되며 항상 초나라부터 시작한다.
- 게임이 시작되면 말이 초기 위치에 세팅된 장기판을 출력한다.
- 장기판의 좌표는 왼쪽 아래부터 가로 0~8, 세로 0~9로 표기한다.
- 입력된 좌표에 따라 장기 말을 움직일 수 있다.
- 움직일 말의 좌표로 상대의 말의 좌표를 입력하면 이동할 수 없다.
- 말이 이동할 위치에 상대의 말이 있으면 잡을 수 있다.
- 장기 말의 움직임 규칙
    - 포를 제외한 모든 기물은 다른 기물을 넘어서 이동할 수 없다.
    - 궁: 모든 방향으로 한 칸씩 이동 가능
    - 사: 모든 방향으로 한 칸씩 이동 가능
    - 차: 전후좌우로 칸 수에 제한없이 이동 가능
    - 포: 전후좌우로 칸 수에 제한없이 이동 가능
        - 단독으로 움질일 수 없고 반드시 1개의 기물을 넘어서 이동해야 한다.
        - 넘는 기물은 상대의 기물이든 자신의 기물이든 상관 없다.
        - 단, 같은 포끼리 서로 넘을 수 없고, 잡을 수도 없다.
    - 마: 앞으로 1칸 이동 후 좌우로 1칸 이동 가능
    - 상: 앞으로 1칸 이동 후 좌우로 2칸 이동 가능
    - 졸/병: 앞으로 또는 좌우로 1칸 이동 가능

## 기능 목록

1. 게임 시작
    - [x] 기물 생성
        1. 초나라/한나라로 팀을 가짐(초나라: 초록색, 한나라: 빨간색)
    - [x] 기물 세팅

    3. [X] 세팅된 기물 출력
2. 초나라 먼저 이동하고자 하는 기물 위치 입력
    1. [X] 입력 예시) 1,1
    2. 입력값 검증
        - [X] split “,”
        - [X] 구분자 없으면 예외 발생
        - [X] 범위 확인 0-8 / 0-9
        - [X] 해당 위치에 말이 있는지
        - [X] 초나라 말인지 확인
3. 해당 기물을 이동 시킬 위치 입력 받기
    - 입력 예시) 2,2
    - 입력값 검증
        - [X] 1,2 같음
        - [X] 보드 범위에 있는지
        - [x] 룰 (해당 말이 이동하는 룰이 맞는지)
        - [x] 경로상에 다른 말이 없는지
        - [x] 종점에 어떤 팀의 기물이 있는지 (적팀이면 가능 / 우리팀이면 불가능)
4. [x] 해당 기물 이동
    - [x] 만약 해당 위치에 적팀 기물이 있다면 제거후 자리 차지
5. [X] 잡았으면 잡았다는 메시지 출력
6. [X] 보드판 출력
7. [X] 한나라 반복
8. 게임 종료 조건
    1. [X] 왕이 죽으면 패배
